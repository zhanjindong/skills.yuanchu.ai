# 代码审查

代码审查是一个自动化代码审查工具。它能够对提交的代码进行全方位审查，识别潜在的 Bug、安全漏洞、性能问题和代码风格问题，并提供具体的改进建议。

## 使用场景

- 提交代码前进行自查，在 Code Review 之前发现和修复常见问题
- 辅助团队 Code Review，提供系统化的检查视角，减少人工遗漏
- 新人入职时作为代码质量教学工具，帮助理解团队最佳实践

## 特点

- **多维度审查**: 覆盖安全性、性能、可维护性、错误处理、命名规范五大维度
- **严重性分级**: 每个问题标注严重程度（Critical / Warning / Suggestion），帮助开发者优先处理关键问题
- **精确定位**: 指出问题所在的具体行号和代码片段，而非泛泛而谈
- **修复建议**: 不仅指出问题，还给出修改后的代码示例
- **语言自适应**: 支持 Java、Python、JavaScript/TypeScript、Go 等主流编程语言的审查规则

## 审查维度

1. **安全性**: SQL 注入、XSS、硬编码密钥、未授权访问等
2. **错误处理**: 未捕获异常、空指针风险、资源泄漏等
3. **性能**: N+1 查询、不必要的内存分配、低效算法等
4. **可维护性**: 重复代码、过长函数、魔法数字、缺少注释等
5. **规范性**: 命名不规范、代码格式、不一致的风格等
