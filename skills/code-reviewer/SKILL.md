# 代码审查 — Claude Code

## 使用方法

在 Claude Code 中输入 `/code-review` 即可触发。可以指定文件路径或让它自动审查暂存区变更。

---

对指定代码进行全方位审查，输出结构化的审查报告。

## 步骤

1. 确定审查范围：
   - 如果用户指定了文件路径，读取该文件
   - 如果未指定，运行 `git diff --cached` 审查暂存区变更
   - 如果暂存区为空，运行 `git diff` 审查工作区变更
2. 逐一分析代码，从以下五个维度进行审查：
   - **安全性**: 检查 SQL 注入、XSS、硬编码密钥、敏感信息泄露、未授权访问
   - **错误处理**: 检查未捕获异常、空值/空指针风险、资源未释放
   - **性能**: 检查 N+1 查询、不必要的循环、低效数据结构
   - **可维护性**: 检查重复代码、过长函数（>50行）、魔法数字、缺少文档
   - **规范性**: 检查命名规范、代码格式、一致性
3. 输出审查报告，格式如下：

```markdown
## 代码审查报告

### 严重问题
- **[安全]** 第 23 行: 存在硬编码数据库密码 → 建议使用环境变量注入
  ```java
  // 修改前
  String password = "admin123";
  // 修改后
  String password = System.getenv("DB_PASSWORD");
  ```

### 警告
- **[错误处理]** 第 45 行: `user.getName()` 可能抛出 NullPointerException → 建议添加空值检查

### 建议
- **[可维护性]** 第 12-67 行: 方法过长（55 行），建议拆分为多个职责单一的方法

### 汇总
- 严重: 1 | 警告: 1 | 建议: 1
```

## 规则

- 每个问题必须给出具体行号
- 严重级别问题必须给出修复后的代码示例
- 不要报告纯风格偏好（如大括号换行风格），除非项目有明确规范
- 如果代码质量良好，也要明确说明"未发现问题"
