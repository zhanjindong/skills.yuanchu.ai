# 测试生成

测试生成是一个自动化单元测试生成工具。它能够分析源代码的业务逻辑，自动生成覆盖正常路径、边界条件和异常场景的单元测试，帮助开发者快速建立测试覆盖。

## 使用场景

- 为已有代码补充单元测试，提升项目的测试覆盖率
- 新功能开发后快速生成测试骨架，再在此基础上补充业务细节
- 重构前先生成回归测试，确保重构不破坏现有功能
- TDD 工作流中快速生成测试用例模板

## 特点

- **场景全覆盖**: 自动识别正常路径、边界条件、异常处理、空值处理等测试场景
- **框架适配**: 根据源代码语言自动选择测试框架（JUnit 5 / pytest / Jest / Go testing）
- **Mock 生成**: 自动识别外部依赖（数据库、HTTP 调用、文件系统），生成对应的 Mock 配置
- **命名规范**: 测试方法采用 `should_预期行为_when_前置条件` 格式，清晰表达测试意图
- **独立运行**: 生成的测试不依赖外部环境，使用内存数据和 Mock 对象

## 生成策略

1. 分析目标函数的输入参数和返回值类型
2. 识别函数内的条件分支（if/else、switch、try/catch）
3. 为每个分支生成对应的测试用例
4. 补充边界值测试（空值、空集合、最大值、最小值）
5. 为外部依赖生成 Mock 和 Stub
