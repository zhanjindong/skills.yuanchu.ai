# 文章翻译

文章翻译工具能把一篇外文文章（输入文章正文或文章网址均可）高质量地翻译成中文，开头标注原文出处，生成排版精良的 HTML 译文页面，并自动推送到 yuanchu.ai/tech/。

不同于机器翻译的逐词硬译，它在忠于原文含义的前提下，按中文表达习惯重组语序、统一专业术语，让译文读起来像中文母语者写的。

## 使用场景

- 翻译技术博客、官方发布公告（如 Anthropic / OpenAI Engineering Blog）
- 翻译外文长文、教程，沉淀到自己的技术站点
- 把英文 PDF / Markdown 文章转成中文 HTML 页面
- 需要保留原文结构（标题、列表、表格、代码块）的整篇翻译

## 特点

- **多源输入**：支持网页 URL、本地 PDF/Markdown/TXT、或直接粘贴正文
- **逐字原文**：网页优先用 curl 抓取原始 HTML 提取正文，避免摘要化导致漏译
- **忠于原文**：完整翻译不删减，专业术语精准且全文一致，圈内常用词保留英文
- **无翻译腔**：按中文习惯重组语序，译文自然流畅
- **结构保真**：标题层级、列表、表格、代码块、引用块一一对应
- **标注出处**：译文开头有原文出处信息卡（原标题 / 作者 / 来源 / 链接 / 日期）
- **自动发布**：生成单文件 HTML 并同步到 yuanchu.ai/tech/，自动更新文章索引

## 输出格式

生成单文件 HTML 译文页面，包含：
- 顶部「译文」标签与中文译题
- **原文出处信息卡**（原标题、作者、来源站点、原文链接、发布日期）
- 译者按语（可选，简述文章主旨）
- 与原文结构一一对应的中文正文
- 底部版权声明（版权归原作者所有，仅供学习交流）

文件命名：`translation-[slug-title].html`（如 `translation-harness-design-long-running-apps.html`）

发布后会同步到 `~/github/yuanchu/yuanchu.ai/tech/`，并在 `tech/index.html` 文章列表顶部插入带「【译】」前缀的条目。
