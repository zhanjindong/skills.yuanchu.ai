# 周报生成 — Claude Code

## 使用方法

在 Claude Code 中输入 `/weekly-report` 即可触发。可以直接从 Git 日志生成，也可以手动输入工作内容。

---

从 Git 提交记录或用户提供的工作内容中，自动生成结构化周报。

## 步骤

1. 确认数据来源：
   - **Git 日志**: 运行 `git log --since="1 week ago" --oneline --author="$(git config user.name)"` 获取本周提交记录
   - **手动输入**: 用户直接描述本周工作内容
   - **混合模式**: Git 日志 + 用户补充
2. 分析并归类工作内容：
   - 按项目或模块分组
   - 区分已完成、进行中、计划中的工作
   - 识别风险和阻塞项
3. 确认周报风格：
   - **简洁版**: 要点式，适合技术团队内部
   - **详细版**: 叙述式，适合跨部门汇报
   - **汇报版**: 面向管理层，突出成果和指标
4. 生成结构化周报，包含本周完成、进行中、下周计划、风险与问题。

## 规则

- 工作项描述具体、有成果导向，避免流水账
- 每项工作聚焦结果而非过程
- 风险和问题要附带建议的解决方案
- 下周计划要具体可衡量
- 默认使用中文输出
- 如果 Git 日志为空，提示用户手动输入工作内容
